# Obsidian Memory System for GitHub Copilot

Persistent memory for GitHub Copilot using an Obsidian vault, Git, and Squad.

Based on Andrej Karpathy's knowledge base pattern: a structured markdown wiki that your AI reads from and writes to across sessions. Each session adds knowledge. Next session starts smarter.

## What this gives you

- Persistent memory across Copilot sessions and projects
- Portable: plain markdown files, works with any AI tool
- Offline access via Obsidian on any device
- Version controlled with full git history
- Curated by a Squad agent team that keeps the vault tidy and linked

## Prerequisites

- Git
- GitHub CLI (gh), authenticated
- Node.js 18+
- Obsidian (https://obsidian.md, free)
- GitHub Copilot subscription (Pro, Pro+, or Enterprise)

### Windows

```powershell
winget install Git.Git
winget install GitHub.cli
winget install OpenJS.NodeJS
winget install Obsidian.Obsidian
npm install -g @bradygaster/squad-cli
```

### macOS

```bash
brew install git gh node
brew install --cask obsidian
npm install -g @bradygaster/squad-cli
```

## Setup

### 1. Create your vault from this template

```powershell
gh repo create my-memory-vault --private --clone --template <owner>/obsidian-memory-template
cd my-memory-vault
```

Replace `<owner>` with the GitHub username hosting this template.

### 2. Initialize Squad

Squad is already scaffolded in this template with three custom agents. Just run:

```powershell
squad init
```

If Squad asks about team setup, the `.squad/` folder already contains the Curator, Archivist, and Librarian agents.

### 3. Open in Obsidian

Open Obsidian. Select "Open folder as vault". Point it at your vault directory.

Install two community plugins (Settings > Community plugins > Turn on > Browse):

- **Obsidian Git**: auto-commits and pushes your vault changes
- **Dataview**: query your notes like a database using YAML frontmatter

### 4. Fill in your global notes

Open these files in Obsidian or your editor and add your info:

- `global/coding-standards.md`: your coding conventions
- `global/tools-and-setup.md`: your development environment
- `global/current-focus.md`: what you are working on right now

### 5. Set GitHub personal instructions

Go to https://github.com/copilot, click your profile picture (bottom left), select Personal instructions. Paste:

```
## Memory Vault (local only)
If you have access to the local filesystem, read my memory vault at <YOUR_VAULT_PATH> before answering project or coding questions.

Start with: AGENTS.md, then index.md, then global/current-focus.md.
For project-specific work, also read wiki/projects/<project-name>/index.md.

This only applies to software engineering, architecture, and project tasks.
Do not load the vault for general conversation, math, or non-technical questions.
```

Replace `<YOUR_VAULT_PATH>` with the full path to your vault.

### 6. Enable Copilot Memory

GitHub.com > Settings > Copilot > Memory > Enable.

This is a complementary cloud layer (per-repo, 28-day TTL, auto-generated by Copilot). It runs in parallel with your local vault.

## Vault structure

```
my-memory-vault/
|-- AGENTS.md                    # Master instructions (Copilot reads this first)
|-- index.md                     # Map of everything in the vault
|-- .github/
|   +-- copilot-instructions.md  # Vault-level Copilot instructions
|
|-- raw/                         # Unprocessed sources (articles, dumps, transcripts)
|-- wiki/
|   |-- patterns/                # Reusable code patterns and anti-patterns
|   |-- decisions/               # Architectural Decision Records
|   +-- projects/                # Per-project memory banks
|       +-- _template/           # Copy this folder for new projects
|-- global/
|   |-- index.md
|   |-- coding-standards.md      # Your coding conventions
|   |-- tools-and-setup.md       # Your environment and tools
|   +-- current-focus.md         # What you are working on right now
|-- curation/
|   |-- index.md
|   +-- curation-rules.md        # Rules for keeping the vault tidy
|-- logs/                        # Session summaries
|-- _inbox/                      # Quick capture, process later
|-- _templates/                  # Note templates
|-- skills/                      # Named workflows
|-- .squad/                      # Squad agent team
+-- .squad-agents/               # Custom agent charters (source of truth)
```

## Using Squad

Squad gives you a team of AI agents that maintain your vault. Start a session:

```powershell
cd <your-vault-path>
copilot --agent squad
```

### Your Squad team

| Agent | Role | What it does |
|-------|------|-------------|
| **Curator** | Vault hygiene | Processes inbox, merges duplicates, fixes broken links, ensures every note has backlinks |
| **Archivist** | Knowledge extraction | Reads raw sources, extracts concepts, creates linked wiki pages |
| **Librarian** | Index and review | Maintains index.md, runs weekly reviews, checks vault health |
| **Scribe** | Session memory | Built-in. Logs decisions and session history automatically. |
| **Ralph** | Watch and triage | Built-in. Autonomous polling and dispatch. |

### Common Squad commands

```
# Ask the Curator to tidy up
"Curator, process everything in _inbox and fix any broken links"

# Ask the Archivist to process raw sources
"Archivist, read everything in raw/ and create wiki pages for each concept"

# Ask the Librarian for a health check
"Librarian, run a vault health check and report orphan pages"

# Ask the whole team
"Team, the vault needs a full curation pass. Process inbox, fix links, update the index."

# Weekly review
"Librarian, run the weekly review"
```

### How Squad maintains links

Every Squad agent enforces linking rules:

- Every note must have a `## Related` section with at least 2 `[[wikilinks]]`
- When creating a new note, agents add backlinks from related existing notes
- The Curator checks for orphan pages (no incoming links) on every curation pass
- The Librarian reports link health stats during weekly reviews

## Feeding memory into the vault

### Quick capture

Drop a note into `_inbox/`. Write whatever format you want. The Curator will process it later: add frontmatter, file it into the right folder, add links.

### Raw sources

Save articles, transcripts, documentation, or any reference material as `.md` files in `raw/`. Then ask the Archivist:

```
"Archivist, process the new sources in raw/ and create wiki pages"
```

The Archivist extracts key concepts, creates one wiki page per concept, adds frontmatter and wikilinks, and updates the index.

### During Copilot sessions

Copilot follows the rules in AGENTS.md. When it learns something reusable during a session, it writes to the vault:

- Patterns go to `wiki/patterns/`
- Decisions go to `wiki/decisions/`
- Project context goes to `wiki/projects/<name>/`

At the end of a session, run the session-close skill:

```
"Run session-close"
```

This summarizes the session, updates wiki pages, and writes a log entry.

### Adding a new project

```
"Run new-project"
```

Or manually:

```powershell
mkdir wiki/projects/my-repo
cp _templates/project-index.md wiki/projects/my-repo/index.md
```

Then add a reference in your project repo's `.github/copilot-instructions.md`:

```markdown
## Memory Vault
Read context from: <YOUR_VAULT_PATH>
Initialize from: AGENTS.md, then index.md, then wiki/projects/<this-repo>/index.md
```

## Skills

| Skill | When to use | What it does |
|-------|-------------|-------------|
| session-close | End of session | Summarize, update wiki, write log |
| curate | Vault feels messy | Process inbox, fix links, deduplicate, update index |
| new-project | Adding a project | Scaffold a project memory bank |
| weekly-review | Friday | Review the week, surface patterns, flag stale notes |

## Note format

Every note uses YAML frontmatter:

```yaml
---
type: pattern | decision | concept | log | rule | project | index
project: repo-name-or-global
status: active | draft | archived
tags: [tag1, tag2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

Body text uses `[[wikilinks]]` for connections. Use descriptive kebab-case filenames: `use-redis-for-session-cache.md` not `untitled-48.md`.

## Redaction rules

Before writing any note, strip:

- Customer names (replace with "Customer A", "a financial services customer")
- Financial instruments (remove entirely)
- Credentials, API keys, tokens (never write)
- PII: emails, phone numbers, employee IDs
- Internal codenames (use public product names)

The agent is the filter. There is no automated pre-commit hook.

---

## Why not Claude Code hooks?

This system was designed for GitHub Copilot, which does not have lifecycle hooks like Claude Code. Instead:

- AGENTS.md instructs Copilot to write knowledge proactively during sessions
- The session-close skill replaces the Stop hook as a manual trigger
- Squad agents handle curation that hooks would automate

If you use Claude Code, you can add hooks on top of this structure. The vault format is tool-agnostic.
